突然间想做一个Blog, 记录一些曾经该记录的东西。20多岁时该做而 又没做的事，如今来做。只能说出来混的，迟早是要还的。Google了一下，意外地发现一种有逼格的写博客的方式 。来本文记录了本Blog（Hexo + Markdown + GitHub）搭建的完整过程。

相关工具

Hexo：一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

Markdown：能让文档更容易读、写和随意改。

GitHub：免费的静态站点，三个特点：免费托管、自带主题、支持自制页面。

发布博客到GitHub

配置GitHub

  • 注册并登录GitHub

  • 访问GitHub Pages ,参照说明创建好 repo

repo的name必须和用户名一致如：brinliu.github.io，首次创建耐心等待10分钟左右审核，之后即可访问静态主页如：http://brinliu.github.io

发布博客

两种方式：

  • 一是通过git工具， 将 blog目录下的 public 中的内容提交到 repo

  • 通过 hexo d 命令来发布, 配置_config.yml文件

必须装好git工具并且配置好环境变量，repository 由于使用https失败改用了git, 此时需要配置ssh-key 配置完成后执行
