做了个小小的Android SDK，往游戏中集成时总遇到点小问题，于是乎，决定学习和了解一下Unity，是为简单的学习笔记。

Environment and Player

设置游戏

创建工程

创建一个3D工程Roll a ball到任意目录

保存场景

一切开始之前先保存场景。 保存时，选择Asserts目录，新建一个Scenes目录，将场景命名为MiniGame

创建游戏面板(游戏区域)

通过Unity Plane来实现。操作：”Game Object - 3D Object - Plane”。
将Plane重命名为Ground, 并且在transform面板中，点设置选择Reset，将此Game Object 在场景的位置设到原点(0,0,0)，把Ground的x,y,z分别缩放到2,1,2。

选择Frame Selected (“Edit - Frame Selected”)，本功能可以方便地切换观察视角，取消选择网格线（”Gizmos - Show Grid”）

创建Player对象

此处使用Unity Sphere来实现。 操作”Game Object - 3D Object - Sphere”。将Sphere重命名为Player. 将Player在场景的位置设到原点(0,0,0).选择Frame Selected，将 Player 移动到Plane之上。所有的Unity对象如objects, cubes, spheres, capsules都有标准的单位尺寸，1x1x1或者1x2x1。因此，此处只需将Y轴位置设为0.5个单位即可。

给背景增加颜色

在Asserts目录下新建一个Materials目录，在该目录下点右键新建一个Material，重命名为Background, 选择这个Material，在Main Maps面板上，点击Albido的颜色拾取器，拾取一个颜色。接下来，把Material Background拖到Scene视图中的Plane之上，完成背景色添加。

调节定向光

在Hierarchy面板选择Directional Light，在Transform面板，把Rotation的Y轴设为60。

移动Player对象

添加物理属性

考虑游戏中的行为，需要为Player对象添加一些物理属性(physics), 选择要关联的Player对象，点击Inspector面板下有个Add Component按钮，选择”Physics - rigidbody”。

通过脚本控制对象运动

在Asserts目录下新建一个Scripts目录，选择要关联的Player对象，点击Add Component按钮，选择New script, 输入CameraController, 创建完成后，将文件移动到Scripts目录。双击打开脚本，进行编辑。

Camera and Play Area

控制Camera移动

通过脚本来控制Camera移动，选择Main Camera对象，点击Add Component按钮，选择New script, 输入CameraController, 创建完成后，将文件移动到Scripts目录。双击打开脚本，进行编辑。

设置游戏区域

在本游戏中，在边沿区设置4面墙即可。创建一个空的Game Object, 命名为Walls, 在外Walls下创建一个Cube, 重命名为West wall, 把West wall的x,y,z分别缩放到0.5,2,20.5。选中West wall，”Edit - Duplicate”, 重命名为East wall, 移动到相应位置，同样创建North wall和South wall。

通过修改脚本控制控制碰撞事件的行为。

####设置Tag

在脚本中判断了Game Object的Tag, 因此需要将Pick Up设置为”Pick Up”。

在Prefabs中选中Pick Up, “Inspector - Tag - Add Tag”,进入Tag manager. 在Tag List中添加一个Tag 0名为”Pick Up”。保存后，在”Inspector - Tag”选择刚才添加的Tag名字。

####优化

static colliders shouldn’t move. Dynamic colliders can move, and have a rigid body attached.Standard rigid bodies are moved using physics forces. Kinematic rigid bodies are moved using their transform.
